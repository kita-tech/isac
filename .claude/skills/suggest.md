# Suggest Skill

現在の状況を分析し、適切なSkillを提案します。

## 使い方

```
/suggest
```

会話の文脈から、今実行すべきSkillを提案します。

## 判断基準

以下の基準で適切なSkillを提案してください：

### /review を提案する場合

- 設計方針について相談されている
- 技術選定で迷っている
- アーキテクチャの決定が必要
- 「どうすべきか」「どちらがいいか」という質問
- 複数の選択肢がある

**キーワード例**: 設計、アーキテクチャ、選定、方針、どちらが、比較、検討

### /code-review を提案する場合

- コードが提示されている
- 実装が完了した報告
- PRを作成する前
- 「見てほしい」「チェックして」という依頼
- バグがないか確認したい

**キーワード例**: レビュー、チェック、見て、実装した、コード、PR、プルリクエスト

### /decide を提案する場合

- 重要な決定が行われた
- 「〜にした」「〜に決めた」という報告
- 技術的な選択が確定した
- 後で参照したい決定事項

**キーワード例**: 決定、決めた、採用、確定、結論

### /memory を提案する場合

- 過去の作業について質問
- 「前に何をしたか」を知りたい
- 類似の問題を過去に解決したか確認
- プロジェクトの経緯を知りたい

**キーワード例**: 前に、以前、過去、履歴、経緯、なぜ

## 出力フォーマット

```
## 💡 Skill提案

現在の状況を分析しました。

### 推奨Skill

**[Skill名]** - [理由]

### 他の選択肢

- [別のSkill] - [こういう場合に有効]

---

実行する場合は `[Skill名]` と入力してください。
```

## 出力例

### 例1: コードが提示された場合

```
## 💡 Skill提案

現在の状況を分析しました。

### 推奨Skill

**/code-review** - コードが提示されているため、品質チェックをお勧めします。

### 他の選択肢

- /review --quick - 設計面も確認したい場合

---

実行する場合は `/code-review` と入力してください。
```

### 例2: 技術選定の相談

```
## 💡 Skill提案

現在の状況を分析しました。

### 推奨Skill

**/review** - 技術選定の相談のため、複数のペルソナで検討することをお勧めします。

### 他の選択肢

- /memory - 過去に同様の決定をしていないか確認する場合

---

実行する場合は `/review [議題]` と入力してください。
```

### 例3: 決定報告

```
## 💡 Skill提案

現在の状況を分析しました。

### 推奨Skill

**/decide** - 重要な決定が行われたため、Memory Serviceに記録することをお勧めします。

---

実行する場合は `/decide` と入力してください。
```

### 例4: 該当なし

```
## 💡 Skill提案

現在の状況を分析しました。

### 結果

特定のSkillは必要なさそうです。このまま会話を続けてください。

### 利用可能なSkill一覧

| Skill | 用途 |
|-------|------|
| /memory | 記憶の検索・管理 |
| /decide | 決定の記録 |
| /review | 設計レビュー（ペルソナ議論） |
| /code-review | コードレビュー（品質チェック） |
| /suggest | Skill提案（このSkill） |
```

## 複数提案する場合

状況によっては複数のSkillを順番に提案：

```
## 💡 Skill提案

現在の状況を分析しました。

### 推奨ステップ

1. **/code-review** - まずコードの品質をチェック
2. **/decide** - 問題なければ実装内容を記録

---

順番に実行することをお勧めします。
```

## 関連スキル

- `/memory` - 記憶の検索・管理
- `/decide` - 決定の記録
- `/review` - 設計レビュー
- `/code-review` - コードレビュー
